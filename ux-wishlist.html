If you only build five things this quarter

  1. T1 OAuth 2.0 done right — system browser, PKCE, Client Creds, per-env token cache, reset-token button.
  2. T2 OpenAPI 2.x + 3.x import (merge, don't clobber).
  3. T3 Prompt-for-variable at send time, with dropdown of prior values.
  4. T0 Save & restore workspace across launches (tabs, env, splits, drafts).
  5. T0 .http file compatibility (VSCode REST Client / JetBrains).

Shipping order

  1. Phase 1 — MVP polish: T0 offline/autosave/workspace-restore, OAuth2 (Client Creds + Auth Code + PKCE), OpenAPI import, cURL paste, prompt-variable, var autocomplete, drag-drop folders, path params, save response examples, Vim mode.
  2. Phase 2 — Pro-ready: Inherit auth w/ override, WebSocket + SSE, HAR import, Postman/Insomnia import, nested envs, recursive vars, reorder env vars, collection-level scripts, configurable shortcuts, plugin API, auto-updater, URL-encode params, perf/load CLI.
  3. Phase 3 — Differentiators: gRPC, MCP client + server, same-named requests, bru.run() orchestration, multipart file vars, SSH/SOCKS5 proxy, HTML doc gen, i18n.

egui-specific UX notes

  • Platform shortcuts must work: Ctrl/Cmd+S, Ctrl+W, Ctrl+T, Ctrl+L — egui apps routinely break these.
  • Bundle a real monospace font (JetBrains Mono, Fira Code). egui's default font ruins JSON readability.
  • Every split boundary persists across launches (Bruno #635 is literally this complaint).
  • Virtualize long lists — sidebar, history, large JSON. ScrollArea handles this but it's easy to forget.
  • Latency is the marketing. Publish cold-start and send-to-first-byte numbers vs Electron competitors.
  • Accessibility: test with a screen reader before release — egui's a11y story is weaker than native, so be deliberate.
